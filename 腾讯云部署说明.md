# BabyLog 腾讯云部署说明

## 当前服务器状态

| 项目 | 内容 |
|---|---|
| 服务器 IP | 不写入仓库；部署时用 `<SERVER_IP>` 代替 |
| 检查日期 | 2026-05-15 |
| SSH `22` | 已从当前 Windows 环境连通 |
| HTTP `80` | 当前未连通 |
| HTTPS `443` | 当前未连通 |
| 当前结论 | 可以先走 SSH 部署；网页访问还需要安装/启动 Nginx，并在腾讯云安全组放行 `80/443` |

## 重要限制

iOS 上要稳定安装 PWA、使用 Service Worker、后续调用相机识别 B 超单，正式环境应使用 **域名 + HTTPS**。

裸 IP 可以用于早期 HTTP 预览，但不建议作为最终入口。

## 服务器一次性初始化

先登录服务器：

```bash
ssh root@<SERVER_IP>
```

确认系统：

```bash
cat /etc/os-release
```

Ubuntu / Debian 可用：

```bash
sudo apt update
sudo apt install -y nginx
sudo systemctl enable --now nginx
```

CentOS / Rocky / AlmaLinux 可用：

```bash
sudo dnf install -y nginx || sudo yum install -y nginx
sudo systemctl enable --now nginx
```

放行系统防火墙：

```bash
sudo ufw allow 80/tcp || true
sudo ufw allow 443/tcp || true
sudo firewall-cmd --permanent --add-service=http || true
sudo firewall-cmd --permanent --add-service=https || true
sudo firewall-cmd --reload || true
```

同时在腾讯云控制台安全组放行：

| 协议 | 端口 | 来源 |
|---|---:|---|
| TCP | `22` | 你的办公/家庭 IP，或临时 `0.0.0.0/0` |
| TCP | `80` | `0.0.0.0/0` |
| TCP | `443` | `0.0.0.0/0` |

## 安装 Nginx 站点配置

在 Windows 项目目录执行：

```powershell
scp .\deploy\nginx-babylog.conf root@<SERVER_IP>:/tmp/babylog.conf
ssh root@<SERVER_IP> "sudo cp /tmp/babylog.conf /etc/nginx/conf.d/babylog.conf && sudo nginx -t && sudo systemctl reload nginx"
```

## 部署前端 PWA

在 Windows 项目目录执行：

```powershell
.\deploy\deploy-pwa.ps1 -HostName <SERVER_IP> -User root
```

如果服务器不是 root 登录，例如 `ubuntu` 用户：

```powershell
.\deploy\deploy-pwa.ps1 -HostName <SERVER_IP> -User ubuntu
```

部署后先访问：

```text
http://<SERVER_IP>
```

## 后续 HTTPS 和 iOS 安装

1. 买一个域名或使用已有域名。
2. 添加 A 记录指向服务器公网 IP。
3. 域名解析生效后，在服务器安装 Certbot。
4. 执行：

```bash
sudo certbot --nginx -d your-domain.example
```

5. iPhone Safari 打开 `https://your-domain.example`。
6. 分享按钮 -> 添加到主屏幕。
