# 育儿期 App 功能缺口调研（Claude）

| 项目 | 内容 |
|---|---|
| 作者 | Claude（Opus 4.7）|
| 日期 | 2026-05-25 |
| 方法 | 调研主流育儿记录 App（亲宝宝 / 宝宝树孕育（出生后）/ 宝贝日志 / 小豆苗 / 国家免疫规划疫苗本 / BabyTracker / Glow Baby 等），按 BabyLog 定位过滤 |
| 性质 | 候选 backlog，**用户挑选后才排期**；非自动进队列 |

## 一、定位过滤（先排除非目标）

BabyLog `非目标` / `DISCLAIMER` 明确不做，故以下主流功能**不纳入缺口**：

- 社区 / 妈妈圈子 / 育儿社交
- 商城 / 母婴电商 / 广告
- 育儿知识资讯 feed
- 早教 / 胎教音乐 / 故事
- AI 育儿问诊
- 第三方账户（微信 / Apple / Google）
- 智能硬件联动（监护器 / 体温贴）
- 宝宝相册社交分享
- 育儿打卡 / 任务 / 游戏化

## 二、BabyLog 现有育儿期能力

baby tab 骨架已在：

- 出生事件结构化（出生日期 / 体重 / 身长头围 等）
- 快捷一拍即记：feed / sleep / diaper / poop / 哺乳 ……
- baby 期日历日选择 + 当日记录
- stage 切换（孕期 → 出生后自动）
- 时间线统一筛选
- 编辑 / 删除 / 详情查看（Q2 / Q2c 已落地）
- 智能录入（语音 + LLM 分类）
- 资料库附件入口
- 家庭同步（S1-S5，附件文件已可跨设备）

## 三、缺口（已按定位过滤，含优先级）

| # | 缺口 | 说明 | 契合度 | 优先 |
|---|---|---|---|---|
| **B1** | **喂养结构化** | 现有"一拍即记"feed 仅打时间戳。缺：母乳 vs 配方奶分类、容量（mL）、时长（min）、哪边乳房（L/R/双）、辅食食材。**家庭高频核心需求**，每天 6-12 次 | 高 | 高 |
| **B2** | **睡眠结构化** | 现有 sleep 仅打时间戳。缺：入睡时间 / 醒来时间 / 自动算时长 / 白觉夜觉区分 / 醒了第几次。家庭高频，新生儿每天 5-8 次 | 高 | 高 |
| **B3** | **尿布结构化** | 现有 diaper 仅打时间戳。缺：尿 / 屎 / 颜色（黄/绿/黑/带血）/ 性状（成型/稀/水）/ 量级。**复诊沟通有用**（医生常问） | 高 | 中 |
| **B4** | **身高体重头围 + WHO 0-5 岁参考百分位曲线** | 类比 FGR。**纯参考、不诊断、不预警**；数据由用户/体检报告录入；曲线对标 WHO 标准（公开数据）。 | 高 | 高 |
| **B5** | **疫苗本（国家免疫规划 + 自费）+ 接种本地提醒** | 中国 0-6 岁免疫规划 14 种疫苗（一类二类）。提醒类同 Q1+Q4 提醒中心 / G7：**纯日期到期通知，非催促、可忽略、不医疗结论**。直击家庭核心刚需 | 高 | 高 |
| **B6** | **体检事件结构化（满月 / 3 / 6 / 9 / 12 / 18 / 24 月 + 之后）** | 类比孕期产检常规层 + screening_*。记录测量值（身高体重头围）+ 体检医生结论 + 下次时间 + 附件。**与 B4 曲线自动联动**（体检自动填入身高体重） | 高 | 中 |
| **B7** | **育儿期复诊汇总导出（baby 版 G9）** | 复用 G9 现有 Markdown 导出 + LLM 润色 fixed-prompt 链路，输出"按时间排列的喂养/睡眠/体检/疫苗摘要"给医生 | 高 | 中 |
| **B8** | **里程碑日记** | 翻身 / 坐 / 爬 / 站 / 走 / 第一声 / 第一颗牙 / 认人 等结构化里程碑。可附照片。**有情感价值**；纯记录、不预设达标年龄、不预警延迟 | 中 | 中 |
| **B9** | **哺乳计时器 / 喂奶间隔提醒** | 类比孕期 G2 宫缩计时器：真计时器 session 开始 / 暂停 / 结束。喂奶间隔提醒可基于上次喂奶时间。**临产新生儿期高频需求** | 中 | 中 |
| **B10** | **用药 / 症状日记** | 退烧药 / 益生菌 / 钙铁锌 服用 + 时长；发烧 / 咳嗽 / 出疹 / 腹泻 等症状记录。**复诊高价值**。同孕期 G5 / G6 模式 | 中 | 中 |
| **B11** | **辅食添加记录** | 6 月+ 添加辅食后专门 track：食材 / 添加日期 / 过敏反应。新食材引入观察期。**纯记录，不预设过敏判定** | 中 | 低 |
| **B12** | **baby 期长文本字段语音录入** | 复用 G11 模式：体检结论 / 症状描述 等长文本字段按住说话直接转写入字段，不走 LLM 分类 | 中 | 低 |

（按 BabyLog 定位**已排除**：宝宝相册社交分享、智能监护器联动、育儿打卡游戏化、AI 育儿建议、辅食食谱推荐、社区社交。这些都属于 `非目标`。）

## 四、医疗安全注记（所有候选通用）

凡涉及"推荐范围 / 区间 / 达标"（B4 WHO 百分位、B5 疫苗时间窗、B11 辅食过敏观察）一律比照 FGR：

- **仅参考**，不诊断、不预警、不催促
- 数值由**用户 / 医院报告**录入，App 绝不计算风险或判读
- 文案与 `DISCLAIMER.md` 一致
- 保人工确认链（save 之前必须用户点保存）
- 疫苗 / 体检提醒措辞中性可忽略，**不医疗结论、不催促**

里程碑（B8）**不预设达标年龄**：不写"翻身正常应在 3 月"这种判断性文案，只让用户记录何时观察到，避免引发家长焦虑。

## 五、建议

- **不建议一次全做**。育儿期是长期主线，按家庭实际使用反馈逐项落地。
- 与孕期 G1-G11 经验对比：**先做高频日常 + 高复诊价值**的项。
- 推荐优先级：**B1 喂养结构化** + **B2 睡眠结构化** + **B5 疫苗本** + **B4 身高体重曲线**，这四项覆盖新生儿 0-12 月主要场景。
- **B3 尿布**、**B6 体检**、**B7 复诊导出**、**B8 里程碑** 等是补完轮。
- **B9 哺乳计时器**、**B10 用药症状**、**B11 辅食**、**B12 长文本语音** 看精力 / 用户实际需要再排。

## 六、与现有同步链路的关系

S1-S5 同步已经覆盖所有 entity 类型（event / attachment / childProfile / familyProfile / familyMember），新增 baby 事件类型**只需要**：

1. 在 `BabyLogDomain.createEvent` 支持新 eventType（如 `baby_feed_v2` / `baby_sleep_v2` / `vaccination` / `body_growth` / `milestone`）
2. 在 `BabyLogFormatters.timelineFilterGroup` 把新类型映射到 `"baby"` 组（避免首页隐身——参考 Q2b-2 跨切硬要求）
3. 在 `isEventVisibleInHome(STAGE_BABY)` 包含
4. 现有 push / pull / 附件链路**全自动覆盖**新类型，无需改同步代码

这意味着育儿期主线化的工程量**主要在 UI + 表单 + 校验 + 文档**，同步层零改动。
