# 孕期 App 功能缺口调研（Claude）

| 项目 | 内容 |
|---|---|
| 作者 | Claude (Opus 4.7) |
| 日期 | 2026-05-19 |
| 方法 | 调研主流孕期 App（宝宝树孕育/美柚/亲宝宝/妈妈网/怀孕管家/育学园 等），按 BabyLog 定位过滤 |
| 性质 | 候选 backlog，**用户挑选后才排期**；非自动进队列 |

## 一、定位过滤（先排除非目标）

BabyLog `非目标`/`DISCLAIMER` 明确不做，故以下主流 App 功能**不纳入缺口**：
社区/孕妈圈子/社交、商城/电商、广告、孕期知识资讯 feed、胎教音乐、专业内容运营（丁香妈妈式）、AI 问诊、第三方账户。

## 二、BabyLog 现有孕期能力

孕周/预产期倒数、产检（常规层结构化；Q2b-2 专项 screening_* 在做）、胎动会话计数器、宫缩快捷记录、B 超指标+OCR+FGR 曲线（≈胎儿估重器）、孕妈指标（体重/血压/血糖，含妊糖软范围）、全局语音录入、统一时间线、备份导入导出、记录编辑、首启免责。

## 三、缺口（已按定位过滤，含优先级）

| # | 缺口 | 说明 | 契合度 | 优先 |
|---|---|---|---|---|
| G1 | **孕期增重管理** | 有体重记录，缺"按孕前 BMI（IOM 标准）给推荐增重区间 + 增重曲线 vs 推荐带"。**参考性，同 FGR 需免责、不诊断、用户/报告录入** | 高 | 高 |
| G2 | **规范宫缩计时器** | 现仅"宫缩快捷记录"。缺真计时器：开始/停止、间隔、持续、频率、规律性与历史。临产高频刚需，几乎所有 App 都有 | 高 | 高 |
| G3 | **孕周/预产期计算器** | 似乎只从"预产期"反推孕周。缺"按末次月经 LMP 推预产期/孕周"+"早期 B 超 CRL 校正预产期"。很多人只知 LMP。基础刚需 | 高 | 高 |
| G4 | **数胎动标准化** | 有会话计数器，缺"每日固定数胎动 + 每天达标（如 2h 内 10 次）用时的历史趋势"。标准 kick-count 模式 | 高 | 中 |
| G5 | **用药/营养素记录** | 叶酸/钙/铁/DHA 等服用记录（可一次记多项）。简单、纯本地、宝宝树该项评分最高 | 高 | 中 |
| G6 | **症状/不适日记** | 孕吐/水肿/出血/头痛/胎动异常等结构化或半结构化记录，复诊沟通价值高；可并入时间线轻量实现 | 高 | 中 |
| G7 | **产检/筛查本地提醒** | Q2b 已有中性待办清单但无本地通知。下次产检/筛查窗口到期本地通知。**措辞须非催促、非诊断；本地通知契合自用、不违非目标** | 中 | 中 |
| G9 | **复诊汇总导出（医生可读）** | 现 JSON 备份仅供 App 重导入，非人/医生可读。本 App 定位即"复诊沟通辅助"，缺一份**按时间排列的 产检/B超/孕妈指标 可读摘要**（文本/可分享/可打印）带给医生。**直击核心用途，现为真缺口** | 高 | 高 |
| G10 | **孕肚照按周记录（bump photo）** | 几乎所有孕期 App 都有（宝宝树孕期相册）；家庭情感价值高、纯本地、非医疗、不违非目标。复用现有附件/图片压缩链路 | 高 | 中 |

（原 G8「待产包/产前准备清单」按用户决定删除，不做。体温记录属小增强，并入 G6 症状日记或孕妈指标，不单列。）

## 四、建议

- **不建议一次全做**；与现有队列（Q2b-2 产检专项、perf、收尾）相比，G1–G3 价值/契合度最高且都是"记录+参考"型、与 FGR/孕妈指标基建可复用。
- 若要排期，建议优先级：**G3 孕周/预产期计算器**（基础、影响所有孕周派生显示与 FGR 准确性）→ **G9 复诊汇总导出**（直击"复诊沟通辅助"核心用途，复用现有事件/时间线数据）→ **G2 宫缩计时器**（临产刚需，独立小模块）→ **G1 增重管理**（复用孕妈指标体重 + 参考带，同 FGR 免责模式）→ G4/G5/G6/G10 视精力。
- 每项立项前单独写需求评估（如产检/语音那样），明确数据模型、是否参考性需免责、与现有事件/时间线/FGR 衔接、医疗安全边界（不诊断、不预警、人工确认）。

## 五、医疗安全注记（所有候选通用）

凡涉及"推荐范围/区间/达标"（G1 增重带、G4 胎动达标、G3 孕周）一律比照 FGR：仅参考、不诊断、不预警、值由用户/报告录入、文案与 `DISCLAIMER.md` 一致、保人工确认。提醒类（G7）措辞中性可忽略，不催促不医疗结论。
