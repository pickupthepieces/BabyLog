## 一、背景与定位

BabyLog 现有 JSON 备份只供 App 自己重导入用，**不是给人/医生看的**。家属带过去复诊，没有可读输出。本项补一个**按时间倒序的可读时序摘要**，可复制/分享/打印带去就诊。

与 Q2c（单条记录详情）不同：Q2c 是"看一条"，G9 是"把一段时期内的产检/B超/孕妈指标/筛查 整合成一份小册子带去给医生"。

## 二、范围与非范围

**范围（v1）**：
- 时间倒序聚合 产检(`pregnancy_checkup`)/B超(`ultrasound`)/孕妈指标(`maternal_metric`)/筛查(`screening_*`)/胎动/宫缩 记录的关键字段，生成 **Markdown 纯文本** 摘要。
- 可在时间范围内筛选（如"全部"/"最近 30/60/90 天"/"自定义起止"）；可选事件类型多选。
- 输出预览页：用户可编辑文本（去掉敏感细节、改措辞），再 **复制 / 系统分享 / 保存为 .md/.txt 文件**。
- **可选 LLM 润色**：用户点按才触发，复用既有 Smart 模型 API 把模板摘要润色成更连贯的医生面文本；原始模板永远是 fallback，未配置 LLM 或失败不影响导出。

**非范围（v1）**：
- 不导出附件原文件（仅在每条标"附件 N 张"提示）；附件直接看走 Q2c → AttachmentPreview。
- 不生成 PDF（系统分享/打印链路可由 Markdown 由用户/系统自行处理）。
- 不入库、不改任何事件/数据模型；汇总是即时生成的临时文本。
- 不自动定时导出；用户主动触发。

## 三、汇总内容（模板示例）

```markdown
# BabyLog 复诊汇总（2026-05-01 至 2026-05-20）
> 仅家庭记录摘要，所有数值/分级由用户或报告原文录入，未经医学判读。本应用非医疗器械。

## 2026-05-18 · 产检（22+6 周·上海一妇婴）
- 血压 118/76 mmHg；体重 60.4 kg；宫高 23 cm；腹围 80 cm；胎心率 142 bpm；尿常规 正常
- 医生结论：胎儿发育正常，下次产检 2026-06-01
- 附件 1 张

## 2026-05-12 · B 超（22 周）
- BPD 53 mm，HC 200 mm，AC 175 mm，FL 38 mm，EFW 540 g
- 备注：羊水正常，胎盘位置 ...
- 附件 2 张

## 2026-05-10 · 孕妈指标
- 体重 60.1 kg；血压 116/74 mmHg；血糖（餐后 1h）7.2 mmol/L

## 2026-05-05 · OGTT（24 周）
- 空腹 4.9；1h 8.6；2h 7.0 mmol/L
- 分级（报告原文）：临界
```

文案规范：
- 顶部固定免责一行，与 `DISCLAIMER.md` 一致风格（"未经医学判读、本应用非医疗器械"）。
- 每条标题：日期 · 类型(GA) · 医院（如有）。
- 每条只列填写过的字段，空字段不显示。
- screening_* 的分级/风险值标注"（报告原文）"，避免被误读为 App 判定。
- 附件只标张数（不嵌图片）。

## 四、可选 LLM 润色（用户点按触发）

**目的**：把模板摘要润色成更自然的医生面叙述；解决"格式像 App 导出"的违和感。

**做法**：
- 预览页底部加一个独立按钮 **"用大模型润色"**（默认不点不发任何数据）。
- 点击后：取当前预览文本 + 一个**固定系统 prompt**（见下）→ 调用既有 Smart 模型 API（与 OCR/智能录入共用配置）→ 返回润色文本 → 替换预览（用户可继续手动编辑或点"恢复原始"还原）。
- **未配置 / 调用失败 / 网络不可用** → 提示并保留原模板，不阻塞导出。

**固定 prompt（医疗安全核心）**：
```
你是一个把家庭孕期记录整理成医生面摘要的助手。严格规则：
1. 仅基于下方给定文本重新组织语序与衔接，使其简洁易读；
2. 禁止添加任何诊断、解释、医学建议、风险判定或新信息；
3. 禁止修改数值、日期、字段值；不会删除字段；
4. 保留原文中的"报告原文"标注；
5. 保留顶部免责声明；
6. 输出纯中文 Markdown，不解释你的工作。
```

**硬约束**：
- 用户点按才发；原文不变前永远可见；产出仅供分享，不入库。
- 数据上行限于"用户即将分享的摘要文本本身"，不额外抓 App 任何其它内容。
- 复用既有隐私文案与 Key 加密本机不出境的纪律（与 OCR/智能录入共用）。
- 润色失败不阻塞导出（原模板可用）。
- **生成结果用户必须可见、可编辑、可还原**——人在回路。

## 五、数据模型 / 接线

- 不改任何事件/数据模型；不新增 prefs；汇总是即时生成的临时文本。
- 复用 `service.listTimelineEvents()`（Q2d 已修正全量）+ 既有 `BabyLogFormatters.eventLabel/formatEventDay/eventSummary` + structured payload 渲染（与 Q2c 详情页同源逻辑可抽取共享 helper）。
- 时间窗筛选在内存进行；事件类型多选靠 `timelineFilterGroup` 复用。

## 六、UI 集成

- 设置/资料库底部加 **"复诊汇总导出"** 入口（settings 或 library 子页 `library/visit-summary` 或 `settings/visit-summary`，体现"非数据库管理而是输出工具"——建议 `library/visit-summary`）。
- 预览页：顶部 时间范围选择 + 事件类型筛选；中部 可编辑预览(Markdown 文本框)；底部 "复制 / 分享 / 保存到本地 / 用大模型润色"。
- 用 NavCompose Screen，AppBar + 返回。

## 七、医疗安全边界（硬约束）

- 模板输出与 LLM 润色后输出**均不得添加诊断/解释/医疗建议**；顶部免责行固定。
- 数值/分级/风险全部带"（报告原文）"或来自 App 内已录字段；App 不计算新判定。
- LLM 润色用户必须看见+可编辑+可还原；不入库。
- 文案与 `DISCLAIMER.md` 一致。

## 八、验收

- 选时间范围内的所有相关事件能正确聚合成 Markdown，倒序、字段无空白行、附件计数正确。
- 不填写时间范围默认"全部"；可只选单类事件。
- "用大模型润色" 用户点按才发；返回结果保留所有数值/字段、不引入新判定；可还原原模板；未配置/失败有中性提示且不阻塞导出。
- 复制 / 系统分享 / 保存文件 全链可用（装机走查）。
- 不创建新事件类型；时间线/FGR/数据模型零回归。
- assemble+lint+smoke 绿；装机走查（含含 screening_* 的混合记录组导出）。

## 九、不做（推迟）

- PDF 直接生成（系统分享/打印可走 Markdown）。
- 自动定时导出 / 云上传。
- 多语言导出（v1 中文）。
