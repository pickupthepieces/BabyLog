## 一、背景与现状

现有 `pregnancy_checkup` 仅浅记录（日期 / 医院 / 结论 + `nextVisitNote` 提取下次日期）。产检是孕期信息量最大的一类，**复杂源 = 每次产检"常规层 + 该孕周专项层"叠加，专项结果是结构化的**，浅文本记录无趋势/复诊价值。本版把它升级为结构化记录。

## 二、标准时间表（仅作"清单提醒"依据，不诊断）

推荐产检孕周：6–13⁺⁶ / 14–19⁺⁶ / 20–24 / 25–28 / 29–32 / 33–36 / 37–41；36 周后每周。

## 三、数据模型

### 3.1 常规层 —— 升级 `pregnancy_checkup` 事件 payload（每次产检容器）

| 字段 | 类型 | 备注 |
|---|---|---|
| occurredAt / 孕周 | 日期 / 自动推算可改 | 复用现有 |
| 医院 / 科室 | 文本 | 可选 |
| 收缩压 / 舒张压 | 数值 mmHg | |
| 体重 | 数值 kg | 可与孕妈指标趋势打通（复用现有，不重复建模） |
| 宫高 / 腹围 | 数值 cm | |
| 胎心率 | 数值 bpm | |
| 尿常规 | 简记（正常/+/++…）或附件 | 不强解析 |
| 医生结论 / 建议 | 长文本 | |
| 下次产检日期 | 日期 | 复用现有 `nextVisitNote` 日期提取逻辑 |
| 附件 | 报告照片 | 复用现有附件机制 |

### 3.2 结构化专项 —— 各自独立轻量事件子类型（便于趋势/编辑/时间线）

按现有 eventType 模型新增子类型（命名示例），每条可独立编辑（复用 Q2 编辑基建）、上时间线：

| eventType | 项目 | 窗口 | 结构化字段（均为用户/报告转录，App 不计算） |
|---|---|---|---|
| `screening_nt` | NT | 11–13⁺⁶ | NT 值 mm；孕周；结论文本(可选) |
| `screening_serum`（唐筛） | 中孕血清学 | 15–20⁺⁶ | 21三体风险值；18三体风险值；开放性神经管风险；分级(低危/临界/高危,选项) |
| `screening_nipt` | 无创 DNA | ~12–22 | T21/T18/T13 低/高风险(选项)；性染色体(可选)；结论 |
| `screening_anomaly` | 大排畸/系统超声 | 20–24 | 结构结论(正常/异常描述,长文本)+附件；不重复 FGR 生物测量(已有 B超录入/FGR) |
| `screening_ogtt` | 糖耐 OGTT | 24–28 | 空腹 / 1h / 2h 血糖 mmol/L；是否异常(用户/报告标,不自动判) |
| `screening_gbs` | GBS | 35–37 | 阴性 / 阳性(选项)；备注 |
| `screening_nst` | 胎心监护 | 32–34起/36后每周 | 反应型 / 无反应型(选项)；备注 |

> 设计原则：① 数值/分级用**结构化字段**才有趋势与复诊价值；② **分级/阴阳由报告或用户填，App 绝不据数值自动判风险/诊断**；③ 大排畸结构判读保持文本+附件，不结构化器官清单（过度且易误导）；④ 不与现有 B超/FGR 生物测量重复建模。

## 四、孕周→专项 清单提醒（纯待办，非诊断）

- 依第二节窗口，给"当前孕周建议关注的专项"**清单式提示**（如 20–24 周提示"系统超声/大排畸"）。
- **必须**：纯待办、可一键忽略/标记已做、措辞为"建议关注"非"必须/异常"；**不得**出现风险判定、异常预警、催促性医疗措辞。中档不做"过期升级/自动报警/FGR 联动"（全档推迟）。

## 五、UI（复用 NavCompose 成果）

- 产检访视 = 一个表单页（复用 P1 `PregnancyCheckup` 表单页升级）：常规层分区 +「本次专项」选择器，选中后展开对应结构化子表单（或引导新建对应 `screening_*` 记录）。
- 各 `screening_*` 可从孕期 quick rail / 智能录入分类进入对应表单页；保存走**显式人工确认**，不自动入库（沿用全局铁律）。
- 时间线按 eventType 渲染；摘要面板可加"最近专项"聚合（复用 `PregnancySummaryPanel`，不改其结构语义）。

## 六、与现有能力衔接

- 复用：附件机制、`nextVisitNote` 日期提取、阶段投影、Q2 编辑基建、智能录入分类（可新增 `screening_*` 候选类型）。
- 不动：FGR/Hadlock 引擎与 B超录入（大排畸不重复测量建模）；孕妈指标体重/血压可与产检常规层数值打通展示但**不重复存储模型**（取舍由 Codex 实现时定，避免双写）。
- OCR：报告照片可走现有多模态识别生成候选，**仍人工确认**；本版不强制，先支持手填+附件。

## 七、医疗安全边界（硬约束）

- App **不计算风险、不判读、不诊断、不预警**；所有分级/阴阳/是否异常均由报告或用户录入。
- 清单提醒为可忽略待办，措辞中性；不得因 App 内容延误就医（与 `DISCLAIMER.md`、首启免责门 Q1 一致）。
- 任何识别结果均候选、人工确认后入库。

## 八、本版不做（推迟到全档/后续）

孕周智能引导自动弹窗、过期升级提醒、与 FGR/时间线深度联动、器官级结构化清单、多胎。

## 九、验收

- 常规层字段齐全可录可改（复用 Q2 编辑）；7 类 `screening_*` 结构化可录、上时间线、可编辑、可趋势（数值类）。
- 清单提醒为可忽略中性待办，无任何诊断/预警措辞。
- 不重复建模 B超/FGR/孕妈指标；保存全程人工确认；assembleDebug+lint+smoke 绿；装机走查孕期录入主链。

## 十、排期建议
