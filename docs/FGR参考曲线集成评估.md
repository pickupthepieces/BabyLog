# FGR 参考曲线集成评估

更新时间：2026-05-18

## 现行决策（2026-05-19）

- 现有 `BabyLogFetalGrowthReference` 只作为“香港近似参考 / 未校准近似引擎”使用，支持家庭记录中的百分位、Z-score 与 P10/P50/P90 参考线展示。
- UI、文档和提示不得称其为正式香港/CUHK FGR 标准、诊断计算或临床级引擎；所有结果仅用于记录和复诊沟通参考。
- OCR 不识别、不推断孕周。孕周只能来自用户手填，或在已有预产期与检查日期时由 App 估算并允许用户修改。
- 后续只有在拿到可追溯文献参数、授权数据或可复核公式后，才能把近似引擎替换为正式参考标准。

## 结论

可以集成。BabyLog 自用长期目标仍是“香港华人人群曲线”，第一轮若缺真实参数，只能显示“香港近似参考”百分位和 Z-score，并必须标注为“未校准近似”，不做诊断结论。

**当前代码状态（2026-05-18 复核）：现有 `BabyLogFetalGrowthReference` 不是正式香港/CUHK FGR 数据引擎。** 代码没有接入 Leung 2008 的完整均值/SD/回归参数，也没有查准香港 EFW 参考方程出处；EFW 部分存在按小程序截图靶值拟合的常数。因此 App UI 与代码 label 必须标为“香港近似参考 / 未校准近似引擎”，不得称为“香港华人人群曲线”或“FGR 诊断计算”。精确版需在拿到可追溯文献参数或授权数据后替换。

原因：

- BPD、HC、AC、FL、EFW 与同孕周参考值对照后，可以得到百分位或 Z 分数，这是胎儿生长评估的常见路径。
- FGR / SGA 相关判断常用 AC 或 EFW 低于第 10 百分位作为参考阈值之一，但不同指南也会使用第 5、第 3 百分位或均数 -2SD，截断值并不完全统一。
- 2019 版中国 FGR 专家共识提到，国内应尽可能选择基于中国人群数据的胎儿生长曲线；其中包括 NICHD 亚裔、基于中国人群的半定制曲线和中国南方人群胎儿生长曲线。用户长期使用的小程序也将曲线选为“香港”，并备注中国南方人推荐采用香港中文大学基于香港人群建立的曲线。
- 当前可直接落地的数据更像“孕周 -> P3/P5/P10/P50/P90/P95/P97”的百分位表。为达到用户现用小程序体验，BabyLog v1 可用 log-normal 近似模型给出 Z-score 和百分位，但 UI/文档必须说明这是未校准近似参考，不是诊断计算。后续若拿到同一标准的均值/SD、LMS 参数或官方公式，再替换为精确 Z-score。

## 推荐第一轮范围

### FGR-1 百分位参考引擎

新增一个只读参考数据模块，支持：

| 输入 | 规则 |
|---|---|
| `metric` | `efwGram` / `bpdMm` / `hcMm` / `acMm` / `flMm` |
| `gestationalAgeDays` | 必须来自用户手动填写的 B 超孕周，或来自预产期计算；不得来自 OCR 推断 |
| `value` | 用户录入或 OCR 生长指标候选确认后的真实数值 |
| `standardId` | 第一轮固定 `hong_kong_chinese`；NICHD / INTERGROWTH / Hadlock 生长标准 / 中国半定制只保留为未来可选，不在当前 UI 暴露 |

输出：

- 对应孕周的 P3/P10/P50/P90/P97 参考值。
- 当前值所在百分位区间，例如 `<P3`、`P3-P10`、`P10-P50`、`P50-P90`、`>P90`。
- 当前值的 Z-score 估算值。
- 参考提示文案，例如“低于第 10 百分位，请带报告咨询医生”，但不得写“诊断 FGR”。

### FGR-2 UI

孕期成长曲线中叠加参考带：

- 默认只显示自有数据线。
- 显示香港近似参考 P10/P50/P90，不做标准切换器；未接入真实参数前不得写成精确香港华人人群曲线。
- 标题或说明展示“香港近似参考（未校准）；仅供记录和复诊沟通参考”。
- 如果当前值低于 P10，首页“待复核”可提示一条“生长指标低于参考百分位，请复核报告并咨询医生”。
- 所有入口必须带“仅供记录和复诊沟通参考，不能替代医生判断”。

### FGR-3 Z-score 分层

v1 做估算 Z-score，满足家庭记录和小程序同级体验；精确 Z-score 后置到拿到同一参考标准的均值/SD、LMS 参数或官方计算公式后。

禁止：

- 把估算 Z-score 包装成诊断级精确计算。
- 把低于 P10 文案写成“FGR”。
- 在缺孕周时计算任何百分位或 Z-score。

## 与 OCR 的关系

OCR v1 不再识别或推断孕周。原因是百分位和 Z-score 对孕周非常敏感，模型推断孕周会直接污染成长曲线和 FGR 提示。

B 超 OCR 只负责候选：

- 检查日期
- BPD
- HC
- AC
- FL
- 明确 EFW

孕周来源：

- 用户按报告手动填写。
- 或在已经维护预产期时，由检查日期和预产期推导，但 UI 必须标记为“按预产期估算”，用户可改。

## 实现顺序

1. `GrowthReferenceStandard` 数据结构和只读查询函数。
2. 在缺真实参数时，只能先落 `hong_kong_chinese` 的近似参考标签；BPD/HC/AC/FL 接香港华人公式口径、EFW 套香港参考这一步必须等可追溯参数查准后再宣称。

### 关键精度陷阱：两个"Hadlock"不能混

- **Hadlock 公式3（EFW 计算式）**：Hadlock FP, et al. Am J Obstet Gynecol. 1985;151:333-337 —— 从 BPD/HC/AC/FL 算 EFW。
- **Hadlock 1991 EFW 生长标准**：上表 Radiology 1991 —— EFW 按孕周的参考带。
- 二者是两回事，不能混用。
- 小程序口径（必须复刻）：**所有标准统一用 Hadlock 公式3 计算 EFW**，再套所选标准的 EFW 均值/SD 算 Z/百分位。**不许**给 INTERGROWTH/WHO 换其原生 EFW 公式——那会和小程序对不上，反而不精确。EFW 一律 Hadlock-3 计算这条写死。

### 后续精确校准靶值（待确认标准选中状态）

用户截图已经给出小程序在“香港”标准下的数值输出。当前版本先实现“香港单标准 + 百分位 + Z-score + 参考线”的体验；精确校准阶段再逐项对齐小程序。

**当前回归靶值：**

| 孕周 | EFW | Z-score | 百分位 |
|---|---|---|---|
| 16⁺⁵ | 134 g | −2.845 | 0.2 |
| 20⁺³ | 344 g | −0.114 | 45.4 |

输入同孕周(精确到天 16周5天 / 20周3天)+EFW，应向小程序数值收敛。BPD/HC/AC/FL 的香港靶值待用户后续从小程序补截（输入同两组测量值切到对应指标读 Z/百分位）后补入。

> 参考对比（仅说明不同标准会给不同结果，**当前版本不实现 NICHD**）：同样 134g@16⁺⁵ 在 NICHD 是 Z−1.657/P4.9，在香港是 Z−2.845/P0.2——香港标准对南方人更严，正是 2019 共识推荐南方人用香港曲线的意义；务必用对香港数据源，别误用 NICHD 数据冒充。

### 后续精确化规则

1. 当前版本不做多标准切换器；只做香港近似参考和近似 Z-score，以达到用户现用小程序的核心体验；UI 必须明确“未校准近似引擎”。
2. 精确香港模式的数据源 = 香港两行一级文献：BPD/HC/AC/FL 用 Leung 2008 UOG；EFW 须先查准 HK EFW 参考方程出处，查不准则不要宣称为精确香港曲线。
3. EFW 一律 Hadlock 公式3（Hadlock FP, et al. Am J Obstet Gynecol. 1985;151:333-337）计算，再套香港 EFW 参考；不混 Hadlock-1991 生长表。
4. 精确模式实现后用本节小程序靶值做数值回归测试。
5. 孕周精确到天；缺孕周不算任何 Z/百分位；不写"诊断 FGR"；常驻"仅供记录和复诊沟通参考，不能替代医生判断"。
6. v1 近似 Z-score 必须在 UI/文档中标注为记录参考；精确 Z-score 必须来自对应标准真实 mean/SD、LMS 或官方公式，禁止误用 NICHD/其他标准数据冒充香港。

### 优先级 / 时序

PREG-07，P1。当前先做小程序同级体验：香港百分位、Z-score、P10/P50/P90 参考线。精确香港曲线校准排在"香港 BPD/HC/AC/FL（Leung 2008）+ 香港 EFW 出处查准 + 授权确认"之后。
