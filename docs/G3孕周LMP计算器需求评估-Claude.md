## 一、背景与定位

当前 BabyLog 的孕周派生完全来自用户填的"预产期"(`expectedDueDate`)，单向反推 GA = 280 − (预产期 − 今天)。现实使用中：
- **大量孕妇只知"末次月经(LMP)"日期**，预产期需要由 LMP 推算；
- **早期 B 超 CRL** 是临床定预产期更准的依据，常用于校正 LMP 推算的 EDD（ACOG/RCOG 标准实践）；
- 用户拿到的产检单上 EDD 可能来自不同来源（自算 / 医生 / B 超），存在差异。

App 缺一个能由 LMP/CRL 推算 EDD 与 GA 的工具。**仅参考；最终预产期由用户/医生确定**——App 不替诊断、不预警差异大小，只如实呈现推算值。

## 二、范围与非范围

**范围（v1）**：
- LMP + 可选 周期长度 → 推算 EDD 与 当前 GA。
- 早期 B 超 CRL（11–13⁺⁶ 周窗口期）→ 推算 EDD（Robinson-Fleming 经典公式）。
- 双源差异提示（中性）：呈现 LMP-EDD 与 CRL-EDD 各自值与差异天数；不替用户判定哪个更准；附中性参考性说明（"临床上差异较大时常以早 B 超为准，请以医生意见为准"）。
- 设置/档案页可独立打开"孕周计算器"工具；新建/编辑预产期时可选"用 LMP 推算"辅助路径。

**非范围（v1）**：
- 不自动覆写 `expectedDueDate`：用户在工具里看到推算结果后，**手动确认**才写入档案；写入仍走现有"编辑档案"链路。
- 不做"差异过大→报警/建议改预产期"——纯展示，不催不预警。
- 不做月经周期不规律情形的智能调整（仅暴露 cycleLength 参数，由用户填）。
- 不动现有 FGR/B 超 OCR/事件模型；CRL 此处仅作为计算工具入参，不创建新事件类型。

## 三、计算公式（公开标准）

### 3.1 LMP → EDD（Naegele 公式 + 周期校正）

```
EDD = LMP + 280 天 + (cycleLength - 28) 天
```

- `cycleLength` 默认 28，UI 允许 21–35 之间用户输入（超出范围弹中性提示"非标准周期，请谨慎使用 LMP 推算"）。
- 推算结果给出 EDD 日期 + 当前 GA（基于 today）。

### 3.2 当前孕周 GA（基于 LMP）

```
GA_days = max(0, today - LMP)
GA_weeks = GA_days / 7, GA_remainder_days = GA_days % 7
```

呈现为"X+Y 周"，最大不超过 42 周（上限处提示"超出常规孕期范围"）。

### 3.3 CRL → GA（Robinson-Fleming 经典公式）

```
GA_days = 8.052 × √(CRL_mm) + 23.73
```

- 公式有效窗口：CRL 约 10–84 mm（对应 ~7⁰–14⁰ 周）；窗口外不给推算（UI 中性提示）。
- 由 CRL-GA 反推 CRL-EDD：`CRL-EDD = exam_date + (280 − GA_days)`。

### 3.4 双源差异

呈现：
- LMP-EDD = ...（含来源标签 "由末次月经推算"）
- CRL-EDD = ...（含来源标签 "由早期 B 超 CRL 推算"）
- 差异 = |LMP-EDD − CRL-EDD| 天
- 中性说明（固定文案，不依差异大小变色变催）："临床上两者差异较大时通常以早 B 超为准，请以您的医生意见为最终依据。"

## 四、数据模型

**v1 推荐：工具结果不入库，避免数据模型扩张**。
- 计算器是 Settings/档案 下的独立工具页；输入 LMP/cycle/CRL 在页面 state 中，离开页面不保留（或仅 `rememberSaveable` 保配置项的临时态）。
- 若用户决定把推算 EDD 写入档案 → 走现有 `expectedDueDate` 编辑路径，**用户主动按"应用为预产期"按钮确认**。

**可选 v2（非本轮，先记录）**：把 LMP 与 cycleLength 存入 `ChildProfile` 作为可选字段，供再次打开计算器/首页显示时复用。当前 `ChildProfile` 不动。

## 五、UI 集成

1. **设置 → 孕周/预产期计算器**（新增 settings 子页 `settings/duedate-calc`，复用 `SettingsPageScaffold`）。
   - 输入：LMP（日期选择器）、cycleLength（默认 28，可选 21–35）；CRL 区块（折叠默认收起）输入 CRL_mm + 检查日期。
   - 输出：LMP-EDD + 当前 GA + CRL-EDD（如有）+ 差异 + 中性说明。
   - 主按钮："应用为预产期"——把选中的 EDD 填入 `expectedDueDate`（弹现有编辑确认或直接跳档案编辑页预填）。
2. **新建孕期家庭 / 编辑档案预产期** 旁加一个小入口"用 LMP 推算"→ 跳上述工具页，结果回填档案表单（仍由用户在档案表单显式保存）。
3. 一个清晰中性 banner："本计算器仅作参考，最终预产期请以医生确认为准。"

UI 复用 NavCompose、`RecordFormScaffold` 不适用（这不是录入）；建议新建轻量 `DueDateCalcScreen.kt`，结构同 `SettingsRootScreen` 的子页。

## 六、医疗安全边界（硬约束）

- 工具仅作算式呈现，**不下结论、不预警、不催促**；中性文案与 `DISCLAIMER.md` 一致。
- 公式有效窗口外不给推算（防误用）；用户填的 cycleLength 超出 21–35 给中性提示但不阻断。
- 应用为预产期需用户在档案表单显式保存（不静默覆写）。
- 不动 FGR/B 超 OCR/事件模型/已锁视觉/阶段投影/密钥/隐私边界。

## 七、验收

- 已知 LMP（如 2026-01-01，cycle=28）→ EDD = 2026-10-08；与权威工具一致。
- cycle=30 同 LMP → EDD = 2026-10-10（+2 天）。
- CRL=50mm，检查日期 2026-04-01 → GA_days ≈ 8.052·√50 + 23.73 ≈ 80.66 ≈ 11⁺3 周；CRL-EDD ≈ 检查日期 + (280−81) ≈ 2026-10-17（与 ACOG 工具一致允许 ±1 天舍入差异）。
- CRL 在窗口外（如 5mm 或 100mm）不给推算，给中性提示。
- 应用为预产期需用户在档案表单按保存才生效；取消则不变。
- 不创建新事件类型；时间线/FGR/数据模型零回归。
- assemble+lint+smoke 绿；装机走查 LMP→EDD、CRL→EDD、双源差异展示、应用为预产期 全链。

## 八、不做（推迟 v2 或后续）

- LMP/cycle/CRL 持久化到 `ChildProfile`（v2）。
- 月经不规律的智能调整（如建议改用 CRL）；保留为用户阅读说明，不机器判定。
- 与 FGR 曲线联动（如以 CRL-GA 重算成长曲线对照点）——本轮独立工具，不联动。
