5 条**全部契合** BabyLog 定位(家庭自用/本地优先/不诊断/复诊沟通辅助/医疗伦理),没有一条偏题。**推荐立项,合理排期**。

## 1. 产检计划 / 提醒 / 日历

**现状**:`pregnancy_checkup` 有 `nextVisitDate`;`PregnancySummaryPanel` 显示"下次产检"倒数。但**这是记录,不是待办**——没有主动提醒、没有"本阶段该做什么 vs 已做"清单、没有"待复核报告"集中视图。

**用户痛点**(NHS 标准:孕妇至少 10 次产检 + 2 次超声 + 各阶段重复血压/尿检/腹围/胎动 等):忘记产检、不知当前阶段该做啥、报告取回延误。

**价值**:🟢 高(直击真痛点)。**契合度**:完全。**复杂度**:中(NavCompose 子页 + 本地通知调度 + 数据聚合 + 待办规则)。

**与既有重叠**:之前 G7"产检/筛查本地提醒"候选,本条是 G7 的**升级版**(加日历视图 + 待办卡 + 未完成报告聚合)。建议吸收 G7 立项一项。

**医疗安全**:
- 提醒文案**中性、非催促、非预警**(如"该做产检了"而非"过期严重")
- 窗口期跨期处理保守(可忽略;不分级危险等级;不替判读)
- 与 Q2b-1 已存的"待复核"标记衔接呈现,不新增 App 判读

**实施要点**:
- 加 `pre_visit_todo` 子结构(可放 ChildProfile prefs 或单独 prefs);用户/系统按孕周生成"建议项"清单,用户可勾选完成/忽略
- 用 Android 本地 `Notification` + `AlarmManager`/WorkManager;权限可拒绝降级
- 列表 UI 复用 RecordFormScaffold;首页加"下次产检卡"明显

## 2. 胎动改"规律观察"(NHS-aligned)

**现状**:`fetal_movement` session counter,可记开始/计数/计时/目标 10 次提示。**问题**:"目标 10 次"暗含阈值判断,与 NHS 立场不一致——NHS 明确"**没有固定每天次数**,重点是了解**宝宝自己的活动模式**,变少/规律改变应立即联系医生"。

**用户痛点**:现行计数器易让用户以"是否达标 10"自评,反而误判。

**价值**:🟢 高(**医疗安全升级**,引用循证)。**契合度**:完全。**复杂度**:小-中(文案重写 + 历史趋势图)。

**与既有重叠**:G4"数胎动标准化"候选 → 本条是 G4 + NHS 校准的升级版,**取代 G4**。

**医疗安全(最关键)**:
- **删除"目标 10 次"暗示**,改为"记录本次会话用时与次数,与你以往模式比较"
- 结果页:**展示历史模式**(过去 N 天每次会话用时/次数趋势)而非"是否达标"
- 文案中添 NHS 通用引用句:"**胎动规律有改变/明显变少时,请立即联系产科或助产士**"(这是引用通用医学共识、非 App 判读,可写)
- **不**做"今日异常"自动判定;不变色不预警

**实施要点**:
- payload 已有 startedAtIso/endedAtIso/count;不动模型
- 结果页 + 历史趋势(横轴日期、纵轴 用时到目标 N 次/每次会话次数)
- 文案全面校准,删旧"目标 10 次提醒"措辞

## 3. 复诊沟通包增强(G9 升级)

**现状**:G9 时序汇总导出可用,但仅做"记录的事件",**缺**:
- "想问医生的问题"清单(用户提前准备)
- "医生嘱咐"(产检中医生说的)
- "待复核 OCR/报告项"集中
- "异常但未判读"原文摘录(B超/筛查软范围)

**用户痛点(NICHD 建议)**:产检前持续记录问题带去问医生,产检后记录医生说的。

**价值**:🟢 高(**直接强化 App 自陈用途"复诊沟通辅助"**)。**契合度**:完全(LibraryRootScreen.kt:56 已有 G9 入口可扩展)。**复杂度**:中。

**实施要点**:
- 新增轻量"产检前问题"事件类型或 ChildProfile 子集 `pre_visit_questions`(用户可随手添加问题,关联到某次产检或独立)
- "医生嘱咐"复用现有 `pregnancy_checkup` 的 `diagnosisText`/医生结论字段,导出 G9 自动聚合
- "待复核":Q2b-1 已有 `reviewCount`,G9 单列一节
- "未判读异常摘录":扫所有 `screening_*`/B超 payload 中有 `(报告原文)` 标记的字段聚合
- **所有摘录加"报告原文"标注**(同 G9 既有规则)

**医疗安全**:不变 — App 不替判读,纯聚合呈现原文。

## 4. 孕期提醒中心(纯用户自定义)

**现状**:无任何主动提醒能力。

**用户主张**:**不做"喝水/补营养/禁忌"这种内容运营**;只做用户自定义提醒(产检/复查/报告取回/药物补剂/胎动观察/备份)。

**价值**:🟢 高(完美贴"本地私人工具"定位,无内容运营风险)。**契合度**:完全。**复杂度**:中-大(本地通知调度 + 权限 + 多反馈;但 Android 标准做法)。

**与 #1 关系**:**强建议合并**——提醒中心是基础设施,#1 的"产检提醒"是它的一个**应用层**。建议结构:
- 底层:统一 reminder system(本地通知 + 调度 + 用户管理)
- 应用层:产检/筛查窗口/胎动观察提示/备份/用户任意自定义

**医疗安全**:用户自定义内容由用户负责,App 仅触发;**不预置任何医疗建议性预设提醒**(如"喝水""叶酸 800mcg")。

**实施要点**:
- 新事件类型 `reminder`(或 prefs 单独存)
- `WorkManager`/`AlarmManager` 调度;Android 13+ POST_NOTIFICATIONS 权限流(可拒绝降级)
- 编辑/列表/启用禁用页面;复用 NavCompose 表单页

## 5. 妊娠结局 / 阶段切换温柔(伦理刚需)

**这条最重要、最敏感、不做有真实风险刺痛用户。** 用户原话点对了一个产品伦理大坑:很多孕期 App 在流产/终止/异常结局上处理粗糙(自动切育儿模式 / 继续推送孕周内容 / 强制安慰文案 / 询问原因等)。BabyLog 当前自动 `currentCareStage`(基于 birthDate/expectedDueDate)**没有"妊娠结束但不育儿"的优雅路径**。

**现状**:`ChildProfile.stageOverride` 已有(PREGNANCY/BABY/UNKNOWN),但缺中间态。

**价值**:🟢🟢 极高(伦理刚需,实现成本不算大)。**契合度**:完全(BabyLog 自陈"尊重家庭、私人工具")。**复杂度**:中。

**实施要点(全文按伦理校准)**:
- `stageOverride` 增加 2 个值:
  - `STAGE_PREGNANCY_ENDED`:**妊娠结束,数据完整保留**,不再 reminder,不切育儿模式
  - `STAGE_PAUSED`:用户主动暂停,所有主动 UI 静音(不显示倒数/不推提醒/不显示"距预产期X天")
- 档案页加"妊娠状态"切换,用户主动选;`stageOverride` 优先于自动判定(spec 既有)
- 切到 `PREGNANCY_ENDED` 时显示**一次性温柔提示**:"我们会保留你的记录;之后想看可以随时回来。不再发提醒,可在档案里再切换。"(中性句、不刺痛、不安慰过度)
- 提醒中心(#1+#4)必须 honor 此 state — `PREGNANCY_ENDED`/`PAUSED` 时**所有自动提醒静音**;用户自定义提醒由用户决定(可继续可关闭,App 不强制)
- 数据保留:**绝对不删任何已记录数据**——可见、可导出、可编辑、可永久访问
- 与 G9 复诊汇总:`PREGNANCY_ENDED` 状态仍可导出全部历史

**伦理硬约束(MUST)**:
- **不询问结局原因**——任何"是流产/引产/出生?"分类一律不做;只一个统一中性状态 `PREGNANCY_ENDED`
- **UI 上不出现"流产/终止/异常"等敏感词**——只说"妊娠结束";图标也不要含"加号医院/心碎"等暗示
- **不做"鼓励再次怀孕"/"复孕指南"/"心理辅导链接"等 vendor-y 内容**——这些是 toC 商业 App 的硬伤
- 不强迫 onboarding 问"是否有过孕产经历";不主动收集敏感词
- 用户主动切回 `PREGNANCY` 也无障碍(不审问"是不是再次怀孕了")

## 推荐排期

按"医疗安全/伦理紧迫度 + 价值/契合 + 与既有依赖关系"排:

1. **#5 妊娠结局温柔(伦理刚需 + 是 #1/#4 的静音 hook 前置)** — 必须先做,否则 #1/#4 的提醒系统设计不全
2. **#2 胎动规律观察(NHS 校准 + 文案重写 + 小趋势图)** — 医疗安全升级,改动小、独立、不依赖其它
3. **#3 复诊沟通包增强(G9 升级 + 问题清单/嘱咐/待复核聚合)** — 复用 G9 框架,直接提升 App 核心用途
4. **#1+#4 合并:孕期本地提醒中心**(基础设施 + 产检应用层)— 体量最大,放最后做整套
5. 既有 G 系列剩余项(G5 用药 / G6 症状日记)看精力

**每项需写独立 eval 才能开工**(像产检/语音/G3/G9 那样):
- 我可以并行写 #2 / #3 / #5 的 eval(都不大,structural fit 清楚)
- #1+#4 合并体量大,需要 reminder 基础架构设计 + UI 流程,单独写时间长一些
- #5 因伦理特殊,eval 时要把所有 MUST 钉死;**不允许任何 vendor 化倾向**

## 不做(明确写下来防漂移)

- 内容运营提醒(喝水/营养建议/禁忌)
- 流产/终止结局询问/分类/原因记录/心理辅导推送
- 任何"是否异常"自动判定(胎动/血压/血糖/筛查皆禁)
- vendor-y 关怀文案/复孕鼓励/SNS 分享
- 任何依赖云端的提醒(本地通知-only)

## 附:#1+#4 合并"本地提醒中心"架构详规

合并理由:#4 是 reminder 基础设施,#1 是它的产检应用层。共一套数据模型与调度器,UI 分两层(产检卡 + 通用提醒)。

### 数据模型

新增独立 prefs 文件 `babylog_reminders.xml`(与既有事件/Smart 配置/Disclaimer 分离,清晰边界);**不进 JSON 导出/家庭同步**(本机偏好,跟设备)。也加入 `babylog_backup_rules.xml` / `babylog_data_extraction_rules.xml` 的 exclude 列。

```
Reminder:
  id: UUID
  kind: enum { CHECKUP_TODO, SCREENING_WINDOW, FETAL_OBSERVATION_HINT, BACKUP, USER_CUSTOM }
  title: String (中性,用户自定义优先;系统生成的默认严格按 spec 文案)
  noteOptional: String
  dueAtIso: String (一次性)| 或 cronLite: String (轻量周期: weekly/monthly/byGestationalWeek)
  source: enum { SYSTEM, USER }
  enabled: Boolean
  dismissedAt: String? (可一键忽略,不再触发)
  completedAt: String? (用户标已做)
  createdAt / updatedAt: String
```

### 调度

- `WorkManager` 周期任务唤醒 + `AlarmManager` setExactAndAllowWhileIdle 单次提醒(Android 6+ Doze 兼容)。
- Android 13+ POST_NOTIFICATIONS 运行时权限(可拒绝降级为页内 List 显示,**不阻塞**)。
- 启动时调度器扫描所有 `enabled && dismissedAt==null && completedAt==null && dueAtIso<=now+window` 的提醒,排上 AlarmManager。
- **#5 状态钩子**:`stageOverride ∈ {PREGNANCY_ENDED, PAUSED}` 时,**所有 SYSTEM 生成提醒静音**;USER_CUSTOM 不强制(由用户决定,可在提醒中心一键全部暂停)。

### UI

- 新增 `tools/reminder-center` 子页(NavCompose):列表 + 新建/编辑/启用禁用/忽略/标已做。
- 首页加"下次产检卡"高亮显示最近 CHECKUP_TODO + 倒数。
- 系统生成的"建议项"(按孕周触发,如 16w-20w 推荐做大排畸)是**默认开,用户可一键忽略**;不是强制清单。
- 触发通知点击 → 跳对应提醒详情/相关记录。

### 系统生成提醒规则(SYSTEM kind)

- **CHECKUP_TODO**:根据 `nextVisitDate`(产检 payload)生成"X 天后产检"提醒(D-3 / D-1 / 当天上午 9 点)。
- **SCREENING_WINDOW**:孕周到达某专项窗口期(如 24w 开 OGTT 窗口)生成中性建议项("如未做可与医生确认是否安排"——不催不预警)。
- **FETAL_OBSERVATION_HINT**:每天上午 X 时(可关闭)中性提示"今天可以观察一下宝宝的活动模式";措辞 NHS 校准。
- **BACKUP**:周期(默认每月)提示"可以做一次本地备份"。

### 文案统一禁区

- 不允许"过期/严重/必须立刻/异常/危险/医生紧急"等词
- 不带评价/不分级危险
- 不主动推送任何"医学建议性"内容(喝水/营养/禁忌)

### 验收

- 创建用户自定义提醒、产检触发后能收到本地通知
- 拒绝通知权限不影响列表/页内展示
- 切到 STAGE_PREGNANCY_ENDED 或 STAGE_PAUSED:所有 SYSTEM 提醒静音(已存 Reminder 仍可见但不触发)
- 备份/同步排除 `babylog_reminders.xml`
- assemble+lint+smoke 绿;装机验通知触发 + 静音 hook
